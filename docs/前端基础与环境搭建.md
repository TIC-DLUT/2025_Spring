## 什么是前端

前端开发，简单来说就是开发用户在浏览器里看到的那部分界面。用户看到的页面、点击的按钮、输入的文本框，都是前端的工作成果。

现代前端开发早已不是"写写 HTML 和 CSS"那么简单了。随着 Web 应用越来越复杂，我们需要框架来帮我们管理代码、组件化开发、处理数据状态。

## 技术栈介绍

本项目使用了以下技术，你不需要一开始就全部掌握，但需要知道它们各自负责什么：

### Vue 3

Vue 是一个用于构建用户界面的 JavaScript 框架。它的核心思想是**响应式**：你定义数据，Vue 自动帮你把数据渲染到页面上，数据变了，页面自动更新。

本项目使用的是 Vue 3，搭配 `<script setup>` 语法糖，写起来非常简洁：

```vue
<template>
  <p>{{ message }}</p>
  <button @click="count++">点了 {{ count }} 次</button>
</template>

<script setup lang="ts">
import { ref } from "vue";

const message = "你好";
const count = ref(0); // ref 创建一个响应式变量
</script>
```

`<script setup>` 是 Vue 3 提供的编译时语法糖，里面定义的变量和函数可以直接在 template 里使用，不需要 return。

### TypeScript

TypeScript 是 JavaScript 的超集，它在 JavaScript 的基础上增加了**类型系统**。有了类型，编辑器就能在你写代码的时候帮你检查错误，而不是等到运行时才发现问题。

```typescript
// JavaScript: 不声明类型，运行时才能发现错误
function add(a, b) {
  return a + b;
}
add("1", 2); // '12'，不是你想要的 3

// TypeScript: 声明类型，写代码时就能发现问题
function add(a: number, b: number): number {
  return a + b;
}
add("1", 2); // 编辑器直接报错！
```

在本项目中，我们用 TypeScript 来定义接口的请求和响应类型，确保前后端数据格式一致。

### Vite

Vite 是一个前端构建工具，它提供了极快的开发体验。我们用 `npm run dev` 启动开发服务器，用 `npm run build` 打包生产环境代码。

Vite 还提供了**代理功能**，可以把前端的 API 请求转发到后端服务器，解决开发时的跨域问题。本项目的代理配置在 `vite.config.ts` 中：

```typescript
server: {
  proxy: {
    '/api': {
      target: 'http://127.0.0.1:8080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, ''),
      // SSE 流式响应需要禁用压缩和缓冲，否则前端收不到逐字输出
      configure: (proxy) => {
        proxy.on('proxyRes', (proxyRes) => {
          if (proxyRes.headers['content-type'] === 'text/event-stream') {
            delete proxyRes.headers['content-encoding'];
            proxyRes.headers['cache-control'] = 'no-cache';
            proxyRes.headers['x-accel-buffering'] = 'no';
          }
        });
      },
    }
  }
}
```

这意味着前端请求 `/api/user/login` 会被代理到 `http://127.0.0.1:8080/user/login`。

`configure` 部分是专门为 SSE 流式响应准备的，它会检测响应的 `content-type`，如果是 `text/event-stream`，就禁用压缩和缓冲，确保 AI 对话的数据能逐字到达前端，而不是被代理缓冲后一次性返回。详细的解释在 [SSE 流式调用](./SSE流式调用.md) 章节中。

### Element Plus

Element Plus 是一套基于 Vue 3 的 UI 组件库，提供了按钮、表单、对话框、菜单等常用组件。我们不需要从零写样式，直接用现成的组件就行。

### Pinia

Pinia 是 Vue 3 的状态管理库。当多个组件需要共享同一份数据时（比如用户登录信息），就需要用它来集中管理。

## Node.js 与 npm

Node.js 是一个 JavaScript 运行环境，npm 是 Node.js 的包管理器。现代前端开发离不开它们。

安装完成后，在命令行输入：

```bash
node -v
npm -v
```

能看到版本号就说明安装成功了。

### npm 常用命令

```bash
npm install        # 安装项目依赖（根据 package.json）
npm run dev        # 启动开发服务器
npm run build      # 打包生产环境代码
```

### package.json

每个前端项目根目录都有一个 `package.json`，它记录了：

- 项目信息（名称、版本）
- 可执行的脚本（dev、build 等）
- 项目依赖（需要安装哪些第三方包）

当你拿到一个新项目，第一件事就是 `npm install` 安装依赖。

## 环境变量

Vite 使用 `.env` 文件管理环境变量，变量名必须以 `VITE_` 开头才能在代码中被读取：

```
# .env.development（开发环境）
VITE_APP_TITLE='AI Chat Dev'
VITE_APP_BASE_API='/api'
```

在代码中通过 `import.meta.env.VITE_XXX` 来访问：

```typescript
const baseURL = import.meta.env.VITE_APP_BASE_API; // '/api'
```

## 启动项目

```bash
# 1. 进入项目目录
cd 2025_Spring

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

确保后端也在运行（Go 服务监听 8080 端口），这样前端才能正常请求后端接口。
