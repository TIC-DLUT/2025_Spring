## 什么是路由

用户在浏览器地址栏输入不同的 URL，看到不同的页面，这个"URL 到页面的映射关系"就是路由。

比如：

- 访问 `/login` → 显示登录页
- 访问 `/AI_Filter/AI_Chat` → 显示 AI 对话页
- 访问了一个不存在的地址 → 显示 404 页面

在传统 Web 开发中，路由是后端负责的（每次访问都刷新整个页面）。但在单页应用（SPA）中，路由由前端负责——页面不会真正刷新，JavaScript 负责切换显示哪个组件。

## Vue Router

Vue Router 是 Vue 官方的路由库。使用步骤：

### 1. 定义路由表

路由表就是"地址和组件的对应关系"：

```typescript
// src/router/routers.ts
export const constantRoute = [
  { path: "/", redirect: "/login" }, // 根路径重定向到登录页
  {
    path: "/login",
    component: () => import("@/views/login/index.vue"), // 懒加载
    name: "login",
    meta: { title: "登录", hidden: true },
  },
  {
    path: "/AI_Filter",
    component: () => import("@/layout/index.vue"), // 外壳是 layout
    redirect: "/AI_Filter/AI_Chat",
    children: [
      // 子路由
      {
        path: "/AI_Filter/AI_Chat",
        component: () => import("@/views/AIFilter/chat/index.vue"),
        name: "AI_Chat",
        meta: { title: "AI对话", icon: "ChatSquare" },
      },
    ],
  },
  {
    path: "/:pathMatch(.*)*",
    redirect: "/404", // 兜底：不认识的地址都跳 404
  },
];
```

几个要点：

**懒加载**：`() => import('...')` 不会在页面打开时就加载所有组件，而是用户访问到那个路由时才加载。这样首页加载更快。

**嵌套路由**：`/AI_Filter` 用了 layout 作为外壳，`children` 里的页面会渲染到 layout 中的 `<router-view>` 位置。这样切换子路由时，侧边栏和顶部栏不会重新渲染。

**meta 字段**：自定义信息，比如 `title` 用于菜单显示，`hidden` 控制是否在菜单中隐藏，`icon` 设置菜单图标。

**兜底路由**：`/:pathMatch(.*)*` 匹配所有未定义的路径，重定向到 404 页面。

### 2. 创建路由器

```typescript
// src/router/index.ts
import { createRouter, createWebHistory } from "vue-router";
import { constantRoute } from "./routers";

const router = createRouter({
  history: createWebHistory(),
  routes: constantRoute,
  scrollBehavior() {
    return { left: 0, top: 0 }; // 切换路由后滚动到顶部
  },
});
```

`createWebHistory()` 使用 HTML5 的 History API，URL 没有 `#` 号，更美观。

### 3. 在 main.ts 中注册

```typescript
import router from "./router/index.ts";
app.use(router);
```

### 4. 在模板中使用

```vue
<!-- App.vue -->
<template>
  <router-view></router-view>
</template>
```

`<router-view>` 是一个占位符，当前路由匹配到的组件会渲染在这里。

## 路由守卫

路由守卫是路由切换时自动执行的钩子函数。你可以把它理解为一个"门卫"——在用户跳转页面前进行检查。

### 为什么需要路由守卫

试想一下：如果用户没登录，直接在浏览器输入 `/AI_Filter/AI_Chat`，会发生什么？

没有路由守卫的情况下，页面会正常显示，但当 AI 对话页发请求时，后端因为没有 token 会返回 401。用户体验很差——都看到页面了才告诉我要登录。

有了路由守卫，用户在跳转之前就会被拦住，直接跳到登录页。

### beforeEach 全局前置守卫

```typescript
// src/router/index.ts
import { GET_TOKEN } from "@/utils/tokens.ts";

router.beforeEach((to, from, next) => {
  const token = GET_TOKEN();

  if (token) {
    // 已登录
    if (to.path === "/login") {
      next("/AI_Filter/AI_Chat"); // 登录页？不需要，去主页
    } else {
      next(); // 正常放行
    }
  } else {
    // 未登录
    if (to.path === "/login") {
      next(); // 去登录页，放行
    } else {
      next("/login"); // 其他页面？先去登录
    }
  }
});
```

三个参数：

- **to**：即将进入的目标路由
- **from**：当前离开的路由
- **next**：放行函数。`next()` 正常放行，`next('/login')` 强制跳转到登录页

逻辑很清晰：

1. 有 token → 想去登录页？不行，给你跳主页去
2. 有 token → 去别的页？没问题
3. 没 token → 想去登录页？去吧
4. 没 token → 想去别的页？先去登录

> 注意：这里只检查了 token 是否存在，没有检查 token 是否过期。更完善的做法是在应用启动时调用后端的 `/user/check` 接口验证 token 有效性。

## 编程式导航

除了在地址栏输入 URL，还可以用代码来跳转页面：

```typescript
import { useRouter } from "vue-router";

const router = useRouter();

router.push("/AI_Filter/AI_Chat"); // 跳转
router.push({ name: "AI_Chat" }); // 按名字跳转
```

在我们的登录逻辑中，登录成功后就是这样跳转的：

```typescript
await userStore.userLogin(form);
router.push("/AI_Filter/AI_Chat"); // 登录成功，跳转到对话页
```
