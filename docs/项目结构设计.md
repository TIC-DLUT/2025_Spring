当我们拿到一个前端项目的时候，第一件事不是看代码，而是看目录结构。一个良好的目录结构能让你快速理解项目在做什么，每个文件负责什么。

先看一下我们项目的目录结构：

```
src/
├── api/             # 接口定义
│   ├── user/        # 用户相关接口
│   │   ├── index.ts # 接口函数
│   │   └── type.ts  # 请求/响应的类型定义
│   └── record/      # 记录相关接口
│       ├── index.ts
│       └── type.ts
├── assets/          # 静态资源（图片、图标等）
│   └── icons/
├── components/      # 全局公共组件
│   ├── index.ts     # 全局组件注册插件
│   └── SvgIcon.vue
├── layout/          # 布局组件
│   ├── index.vue    # 布局主组件
│   ├── logo/        # Logo 区域
│   ├── menu/        # 侧边菜单
│   ├── tabbar/      # 顶部导航栏
│   │   ├── breadcrumb/ # 面包屑
│   │   └── setting/    # 设置按钮
```

## 为什么这样分

可能你会觉得，直接把所有代码写在一个文件里不就行了？确实可以，但当项目变大后，你会完全找不到东西在哪。

分目录的核心思想是**各司其职**：

- **api/**：负责和后端通信。后端有多少接口，这里就有多少函数。把接口调用集中管理，改接口的时候不用翻遍整个项目找 `axios.post`
- **views/**：负责页面。用户看到的每一个"页面"对应一个文件夹
- **components/**：负责可复用的小组件，多个页面都可能用到的东西放这里
- **layout/**：负责整体布局框架（侧边栏 + 顶部栏 + 内容区）
- **store/**：负责全局共享的数据。比如用户登录状态，多个页面都需要知道
- **router/**：负责"地址和页面的对应关系"。用户访问 `/login` 显示登录页，访问 `/AI_Filter/AI_Chat` 显示对话页
- **utils/**：负责通用工具函数，哪里都可能用到的辅助代码
- **styles/**：负责全局样式和 CSS 变量

## api 层的设计

后端的每个接口，我们在前端都有一个对应的函数。比如后端有 `POST /user/login`，前端就有：

```typescript
// src/api/user/index.ts
export const reqLogin = (data: LoginForm): Promise<LoginResponse> =>
  request.post(API.LOGIN_URL, data);
```

同时，在 `type.ts` 中定义好请求和响应的类型：

```typescript
// src/api/user/type.ts
export interface LoginForm {
  telephone: string;
  password: string;
}

export interface LoginResponse {
  code: number;
  token: string;
  message: string;
}
```

这样做的好处是：

1. 后端接口改了，只需要改 api 层，不用改业务代码
2. TypeScript 类型让你写代码时有自动补全，不会写错字段名
3. 新人看 api 目录就知道项目有哪些接口

## views 和 layout 的关系

`layout/` 是整个应用的"骨架"：左侧菜单 + 顶部导航 + 右侧内容区。而 `views/` 中的页面组件会被渲染到右侧的内容区里。

这通过路由的嵌套来实现：

```typescript
{
  path: '/AI_Filter',
  component: () => import('@/layout/index.vue'),  // 使用 layout 作为外壳
  children: [
    {
      path: '/AI_Filter/AI_Chat',
      component: () => import('@/views/AIFilter/chat/index.vue'),  // 页面内容
    }
  ]
}
```

`layout/index.vue` 中有一个 `<router-view>`，子路由对应的页面组件就渲染在这个位置。
