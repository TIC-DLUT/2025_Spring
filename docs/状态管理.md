## 为什么需要状态管理

先想一个场景：用户在登录页登录成功后，token 被保存到了 localStorage。在对话页，需要用这个 token 去请求 AI 接口。

问题来了：登录页和对话页是两个独立的组件，它们怎么共享"用户已登录"这个信息？

最笨的办法是每个组件都从 localStorage 读一遍。但 localStorage 只能存字符串，读写不方便，而且不是响应式的——数据变了，页面不会自动更新。

所以我们需要一个**集中管理共享数据的地方**，这就是状态管理。

## Pinia

Pinia 是 Vue 3 官方推荐的状态管理库（Vue 2 用 Vuex）。它的用法非常直观：

```typescript
import { defineStore } from "pinia";

const useCounterStore = defineStore("counter", {
  state: () => ({
    count: 0,
  }),
  actions: {
    increment() {
      this.count++;
    },
  },
  getters: {
    doubleCount: (state) => state.count * 2,
  },
});
```

三个核心概念：

- **state**：存储数据的地方（相当于组件的 data）
- **actions**：修改数据的函数（可以包含异步操作）
- **getters**：基于 state 计算出的值（相当于 computed）

在组件中使用：

```typescript
const counterStore = useCounterStore();

counterStore.count; // 读取
counterStore.increment(); // 调用 action
counterStore.doubleCount; // 使用 getter
```

## 本项目的 Store 设计

### 用户 Store（user.ts）

这是最核心的 store，管理用户登录状态：

```typescript
import { defineStore } from "pinia";
import { SET_TOKEN, GET_TOKEN, REMOVE_TOKEN } from "@/utils/tokens.ts";
import { constantRoute } from "@/router/routers.ts";
import { reqLogin, reqRegister } from "@/api/user";

let useUserStore = defineStore("User", {
  state: () => ({
    token: GET_TOKEN(), // 从 localStorage 恢复 token
    menuRoutes: constantRoute, // 菜单所需的路由数组
  }),

  actions: {
    async userLogin(data: LoginForm) {
      let result = await reqLogin(data);
      if (result.code == 0) {
        this.token = result.token;
        SET_TOKEN(this.token); // 存到 localStorage
        return "ok";
      } else {
        return Promise.reject(new Error(result.message));
      }
    },

    async userRegister(data: LoginForm) {
      let result = await reqRegister(data);
      if (result.code == 0) {
        return "ok";
      } else {
        return Promise.reject(new Error(result.message));
      }
    },

    // 退出登录：清除 store 和 localStorage 中的 Token
    logout() {
      this.token = "";
      REMOVE_TOKEN();
    },
  },
});
```

看一下登录的完整流程：

1. 用户在登录页点击"登录"按钮
2. 组件调用 `userStore.userLogin(form)`
3. store 调用 `reqLogin` 发起 HTTP 请求
4. 后端返回 `{ code: 0, token: "xxx" }`
5. store 把 token 存到 state 和 localStorage 中
6. store 返回 `'ok'`，组件执行 `router.push` 跳转到对话页
7. 之后的请求，request 拦截器会自动从 localStorage 取 token 带上

如果登录失败（code 不是 0），store 会 `Promise.reject`，组件的 catch 就能捕获到错误，显示提示。

### Token 存取工具

```typescript
// src/utils/tokens.ts
export const SET_TOKEN = (token: string) => {
  localStorage.setItem("TOKEN", token);
};
export const GET_TOKEN = () => {
  return localStorage.getItem("TOKEN") ?? "";
};
export const REMOVE_TOKEN = () => {
  localStorage.removeItem("TOKEN");
};
```

为什么不直接用 `localStorage.setItem`，而是封装成函数？

1. 统一 key 名，不会写错
2. 以后如果想改成 Cookie 存储，只改这一个地方
3. `?? ''` 确保没值时返回空字符串而不是 null

### 布局 Store（setting.ts）

```typescript
let useLayoutStore = defineStore("SettingStore", {
  state: () => ({
    fold: false, // 侧边栏是否折叠
    refresh: false, // 是否正在刷新
  }),
});
```

这个 store 管理布局相关的状态。当用户点击折叠按钮时，`fold` 变为 true，layout 中侧边栏变窄，内容区变宽。这个状态需要多个组件共享（菜单、tabbar、内容区都需要响应 fold 的变化），所以放到 store 中。

## Store 的数据流

```
组件 → 调用 action → action 修改 state → state 变化 → 组件自动更新
         ↓
      发起 HTTP 请求
         ↓
      后端返回数据
         ↓
      存到 state + localStorage
```

单向数据流，清晰可追踪。
