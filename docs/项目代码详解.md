## 学习说明

本章将按照数据流的顺序，从入口文件开始，逐个讲解项目中每个文件的作用和实现细节。

代码已经实现了注释覆盖，直接阅读代码也可以。本篇文档帮助大家建立全局视角，理解各模块如何串联。

## 入口：main.ts

```typescript
import { createApp } from "vue";
import App from "./App.vue";
import ElementPlus from "element-plus";
import "element-plus/dist/index.css";
import zhCn from "element-plus/es/locale/lang/zh-cn";
import "virtual:svg-icons-register";
import router from "./router/index.ts";
import pinia from "@/store/index.ts";
import gloablComponent from "./components/index.ts";

const app = createApp(App);

app.use(pinia);
app.use(ElementPlus, { locale: zhCn });
app.use(router);
app.use(gloablComponent);

app.mount("#app");
```

main.ts 做的事情就是"安装插件，挂载应用"：

1. **createApp(App)**：创建 Vue 应用实例
2. **pinia**：安装状态管理
3. **ElementPlus**：安装 UI 组件库，`zhCn` 设置中文语言包
4. **router**：安装路由
5. **gloablComponent**：注册全局组件（SvgIcon 等）
6. **mount('#app')**：把应用挂载到 index.html 中的 `<div id="app">`

## 根组件：App.vue

```vue
<template>
  <router-view></router-view>
</template>

<script setup lang="ts"></script>

<style scoped></style>
```

非常简洁——只有一个 `<router-view>`。它就是路由的出口，当前匹配到的路由组件会渲染在这里。

## 布局：layout/index.vue

```vue
<template>
  <div class="layout_container">
    <div class="layout_slider" :class="{ fold: layoutSettingStore.fold }">
      <Logo />
      <el-scrollbar>
        <el-menu :collapse="layoutSettingStore.fold">
          <Menu :menuList="userStore.menuRoutes" />
        </el-menu>
      </el-scrollbar>
    </div>
    <div class="layout_tabbar" :class="{ fold: layoutSettingStore.fold }">
      <Tabbar />
    </div>
    <div class="layout_main" :class="{ fold: layoutSettingStore.fold }">
      <Main />
    </div>
  </div>
</template>
```

布局分为三个区域：

- **layout_slider**：左侧菜单栏，宽度 `@menu-width`（在 variables.less 中定义）
- **layout_tabbar**：顶部导航栏，包含面包屑和设置按钮
- **layout_main**：右侧内容区，子路由的页面组件渲染在这里

当 `layoutSettingStore.fold` 为 true 时，三个区域都会添加 `fold` class，侧边栏变窄，内容区变宽。

`:class="{fold: layoutSettingStore.fold}"` 是 Vue 的动态 class 绑定语法——当 fold 为 true 时，添加 fold 这个 class。

## 菜单组件：layout/menu/index.vue

```vue
<template>
  <template v-for="item in menuList" :key="item.path">
    <!-- 无子路由 -->
    <el-menu-item
      v-if="!item.children && !item.meta.hidden"
      :index="item.path"
      @click="goRoute(item)"
    >
      <el-icon><component :is="item.meta.icon" /></el-icon>
      <template #title
        ><span>{{ item.meta.title }}</span></template
      >
    </el-menu-item>

    <!-- 有子路由但只有一个 -->
    <el-menu-item v-if="item.children && item.children.length === 1" ...>
      ...
    </el-menu-item>

    <!-- 有多个子路由 -->
    <el-sub-menu v-if="item.children && item.children.length > 1" ...>
      <Menu :menuList="item.children" />
      <!-- 递归 -->
    </el-sub-menu>
  </template>
</template>
```

这个组件根据路由表动态生成菜单，支持三种情况：

1. 没有子路由 → 直接显示一个菜单项
2. 只有一个子路由 → 也直接显示（避免只有一个子项还要展开的尴尬）
3. 多个子路由 → 显示一个可展开的子菜单

`<component :is="item.meta.icon" />` 是 Vue 的动态组件，根据路由 meta 中配置的 icon 名称渲染对应的 Element Plus 图标。

注意最后一行：`<Menu :menuList="item.children" />` ——这个组件**递归调用自己**来处理多级菜单。

## 登录页：views/login/index.vue

登录页的核心逻辑：

```typescript
const handleSubmit = async () => {
  // 1. 表单验证
  try {
    await loginForms.value?.validate();
  } catch (error) {
    return; // 验证没过，直接返回
  }

  // 2. 调用登录
  try {
    loading.value = true;
    await userStore.userLogin(form);
    router.push("/AI_Filter/AI_Chat");
    loading.value = false;
    ElNotification({ message: `登录成功,${getTime()}`, type: "success" });
  } catch (error: any) {
    loading.value = false;
    ElNotification({
      title: "登录失败",
      message: error.message,
      type: "warning",
    });
  }
};
```

`el-form` 的 `validate()` 方法会根据 `rules` 规则验证表单。我们的规则很简单——两个字段都必填：

```typescript
const rules = reactive({
  telephone: [{ required: true, message: "请输入电话号码", trigger: "blur" }],
  password: [{ required: true, message: "请输入密码", trigger: "blur" }],
});
```

注意 el-form-item 的 `prop` 属性必须和 form 对象的字段名一致，validate 才能正确工作。

## AI 对话页：views/AIFilter/chat/index.vue

这是项目中最复杂的组件，核心点：

1. **消息列表**：`messages` 数组存储所有对话消息，每条消息有 role（user/assistant）、content、time
2. **Markdown 渲染**：`formatMessage` 用 marked + DOMPurify 处理 AI 回复
3. **SSE 流式调用**：`sendMessage` 中通过原生 `EventSource` + Cookie 传递 Token 建立 SSE 连接，逐字追加 AI 回复
4. **自动滚动**：`watch(messages, scrollToBottom, { deep: true })` 消息变化时自动滚到底部
5. **打字动画**：`isLoading` 为 true 时显示三个跳动的圆点

详细的 SSE 和 Markdown 部分请参阅对应的专题文档。

## Token 工具：utils/tokens.ts

```typescript
export const SET_TOKEN = (token: string) => {
  localStorage.setItem("TOKEN", token);
};
export const GET_TOKEN = () => {
  return localStorage.getItem("TOKEN") ?? "";
};
export const REMOVE_TOKEN = () => {
  localStorage.removeItem("TOKEN");
};
```

简单的 localStorage 封装，统一管理 Token 的存取与清除。

## 样式系统：styles/

### variables.less

定义全局 Less 变量：

```less
@menu-width: 210px;
@menu-fold-width: 50px;
@menu-bg-color: #283646;
@menu-logo-height: 50px;
@top-height: 50px;
@top-bg-color: #fff;
@content-bg-color: #f5f5f5;
```

这些变量在 `vite.config.ts` 中通过 `additionalData` 自动注入到每个 Less 文件中，所以任何 `.vue` 文件的 `<style lang="less">` 里都可以直接使用 `@menu-width` 等变量，不需要手动 import。

### scrollBar.less

自定义滚动条样式，让滚动条变细变好看。

## 全局组件注册：components/index.ts

```typescript
import SvgIcon from "./SvgIcon.vue";
import type { App } from "vue";

export default {
  install(app: App) {
    app.component("SvgIcon", SvgIcon);
  },
};
```

实现了一个 Vue 插件，在 main.ts 中 `app.use(gloablComponent)` 时自动触发 install 方法，把 SvgIcon 注册为全局组件。之后在任何组件中都可以直接 `<SvgIcon />` 使用，不需要单独 import。
